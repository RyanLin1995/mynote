## 1. 文件基本说明
范例：`drwxr-xr-x.   2    ryan   ryan    6      7月 11 10:09   Desktop`

* `drwxr-xr-x.`：代表文件的类型和权限
1. 第一个字符代表文件的类型，有：
	* [d]表示目录
	* [-]表示文件
	* [l]表示链接档
	* [b]表示装置文件里面的可供储存的接口设备(可随机存取装置)
	* [c]装置文件里面的串行端口设备，例如键盘、鼠标(一次性读取装置)
2. 接下来的字符中，以三个为一组，且均为『rwx』 的三个参数的组合。其中，[ r ]代表可读(read)、[ w ]代表可写(write)、[ x ]代表可执行(execute)
	* 第一组为文件拥有者可具备的权限，即owner权限
	* 第二组为加入此群组账号的权限，即group权限
	* 非本人且没有加入本群组之其他账号的权限，即other权限

* `2`：代表连接数
每个文件都会将他的权限与属性记录到文件系统的 i-node 中。不过，我们使用的目录树却是使用文件名来记录，因此每个档名就会连结到一个 i-node 。这个属性记录的，就是有多少不同的档名连结到相同的一个 i-node 号码

* `ryan`：这个文件(或目录)的拥有者账号
* `ryan`：这个文件的所属群组
在 Linux 系统下，你的账号会加入于一个或多个的群组中
* `6`：文件的容量大小，默认单位为 bytes
* `7月 11 10:09`：文件的建档日期或者是最近的修改日期
如果想显示完整时间，可以用 `ls -al --full-time` 或 `ll -a -full--time`
* `Desktop`：为文件的档名
档名之前多一个 [.] ，则代表这个文件为隐藏档

## 2. 文件或目录时间的意义
* modification time (mtime)（默认显示）:
当文件【内容】变更时，就会更新这个时间。即当初的文件里面的内容发生了变更

* status time (ctime):
当文件【状态】变更时，就会更新这个时间，即文件的权限或属性发生了变更

* access time (atime):
当文件内容被读取时，就会更新这个时间，即文件被访问(如利用 `cat`去读文件)

PS:如果只通过vim访问文件，只修改atime，如果通过vim读取并修改文件，则所有时间都会被更改

PPS：如果想修改 mtime、atime或 ctime，可以使用 `touch` 命令

## 3. 文件与目录权限的意义
|组件|内容|迭代物件|r|w|x|
|-|-|-|-|-|-|
|文件|文本文件、数据库内容文件、二进制可执行文件等|文件文件夹|读取文件内容|修改文件内容|执行文件|
|目录|记录文件列表|可分类抽屉|读取文件名|修改文件名|进入该目录的权限|
* PS: 对于只有`r`而没有`x`的目录，是只能看到目录里面的文件名称而无法进入(cd)该目录的
* PPS: 一个是你为owner且具有`rwx`权限的目录，如果里面有一个owner跟group都与你无关的且只有`r`权限的文件，你是可以删除掉的。因为你对于这个目录具有`rwx`的权限

## 4. 文件预设权限
`umask` 或 `umask -S`: 指目前用户在建立文件或目录时候的权限默认值。
  * 建立文件时：因为建立文件一般是没有可执行权限(x)的，因此一般文件权限最大分数为666，即：
	`-rw-rw-rw-`
  * 建立目录时：因为建立目录一般需要可执行权限进入该目录，因此一般目录权限最大分数为777，即：
	`drwxrwxrwx`

`umask`的分数指的是默认值需要减掉的权限，如`umask` = 002，则:
* 新建文件时:(-rw-rw-rw-) - (--------w-) = (-rw-rw-r--)
* 新建目录时:(drwxrwxrwx) - (d-------w-) = (drwxrwxr-x)

PS: root 与 user 的预设`umask`并不相同, root的预设umask为022,, user预设`umask`为002

要设定`umask`,只需在`umask`后面接上要减去的权限数字即可，例如修改`umask`从022为002: `umask 002`

## 5. 绝对路径与相对路径
* 绝对路径(absolute path)：由根目录(/)开始写起的文件名或目录名称， 例如 `/home/dmtsai/.bashrc`
* 相对路径(relative path)：相对于目前路径的文件名写法。 例如 `./home/dmtsai` 或 `../../home/dmtsai/` 等等。反正开头不是`/`就属于相对路径的写法
* `.`: 代表当前的目录，也可以使用 `./` 来表示
* `..`: 代表上一层目录，也可以 `../` 来代表
* `-`: 代表前一个工作目录
* `~`: 代表当前用户的家目录
* `~account`: 代表account这个用户的家目录

## 10. 文件的特殊权限
### Set UID: SUID
状态: `-rwsr-xr-x`
当一个可执行文档的拥有者，其执行权限 x 上出现 s 时，即设置了SUID特殊权限。当用户使用有这个特殊权限的二进制文件时，用户会暂时获得该文件拥有者的权限

SUID的基本功能与限制：
* SUID 权限仅对二进制程序(binary program)有效；
* 执行者对于该程序需要具有 x 的可执行权限；
* 本权限仅在执行该程序的过程中有效 (run-time)；
* 执行者将具有该程序拥有者 (owner) 的权限。

例子：`/usr/bin/passwd`命令
1. user 对于 /usr/bin/passwd 这个程序来说是具有 x 权限的，表示 user 能执行 passwd；
2. passwd 的拥有者是 root 这个账号；
3. user 执行 passwd 的过程中，会【暂时】获得 root 的权限；
4. /etc/shadow 就可以被 user 所执行的 passwd 所修改。

PS：SUID 仅可用在 binary program 上


### Set GID: SGID
状态: `-rwxr-sr-x`
当一个目录执行权限 x 上出现 s 时，即设置了SGID特殊权限。SGID可用于文件与目录。
* 用于文件时：
  * SGID 对二进制程序有用；
  * 程序执行者对于该程序来说，需具备 x 的权限；
  * 执行者在执行的过程中将会获得该程序群组的支持

例子: `/usr/bin/locate`
`/usr/bin/locate`命令的群组为 slocate , 当 user 去执行 locate 时, usr 将会取得 slocate 群组的支持，因此就能够去读取群组同样为 slocate 的 mlocate.db文件

* 用于目录时:
  * 用户若对于此目录具有 r 与 x 的权限时，该用户能够进入此目录；
  * 用户在此目录下的有效群组(effective group)将会变成该目录的群组；
  * 用途：若用户在此目录下具有 w 的权限(可以新建文件)，则使用者所建立的新文件，该新文件的群组与此目录的群组相同。

### Sticky Bit: SBIT
状态: `-rwxrwxrwt`
当一个目录执行权限 x 上出现 t 时，即设置了SBIT特殊权限。
SBIT仅对目录作用：
* 当用户对于此目录具有 w, x 权限，亦即具有写入的权限时；
* 当用户在该目录下建立文件或目录时，仅有自己与 root 才有权力删除该文件

例子：
当甲这个用户于 A 目录是具有群组或其他人的身份，并且拥有该目录 w 的权限，这表示【甲用户对该目录内任何人建立的目录或文件均可进行 "删除/更名/搬移" 等动作】。不过，如果将 A 目录加上了 SBIT 的权限项目时，则甲只能够针对自己建立的文件或目录进行删除/更名/移动等动作，而无法删除他人的文件。

### SUID、SGID、SBIT权限设定
与 `rwx` 相似，SUID、SGID、SBIT的数字权限分别为4/2/1。亦可使用 `ugo+-=` 的模式
例子：
1. 给权限为 rwxr--r-- 的文件加入SUID权限，即 rwsr--r--
   `chmod 4744 filename` 或 `chomd u=rwxs,g=r,o=r filename`
2. 给权限为 rwxr-xr-x 的目录加上SUID和SGID权限，即 rwsr-sr-x
   `chomd 6755 filename` 或 `chmod u=rwxs,g=rxs,o=rx filename`
3. 给权限为 rwxrwxrwx 的目录加上SBIT权限，即 rwxrwxrwt
   `chomd 1777 filename` 或 `chomd u=rwx,g=rwx,o=rwt filename`

PS: 如果文件或目录本身不具备执行权限，但是加上了SUID/SGID/SBIT，即加上了空的权限，则 S 跟 T 会变为大写

## 11. 文件的查找

### 查找执行文件: which ; type
#### which: 
用法: `which [-a] scripname`

|选项与参数|说明|
|-|-|-|
|-a|所有由 PATH 目录中可以找到的指令均列出，而不止第一个被找到的指令名称|

* PS: `which` 只会根据【PATH】这个环境变量所规范的路径，去搜寻『执行档』的档名，因此无法搜索到bash内置命令如`history`

#### type:
用法: `type -[ap] scripname`

|选项与参数|说明|
|-|-|-|
|-a|显示所有包含名称为 scripname 的可执行文件的位置|
|-p|为每个 NAME 名称进行 PATH 路径搜索，即使它是别名或内建函数，并且返回磁盘上文件的名称|

* PS: `type` 可以列出所有的执行文件

### 由一些特定的目录中寻找文件或文件名: whereis
用法: `whereis [-bmsu] 文件名或目录`

|选项与参数|说明|
|-|-|-|
|-l|列出 whereis 会去查询的几个主要目录|
|-b|只找出binary格式的文件(只找bin文件夹)|
|-m|只找出说明文件(只找man文件夹)|
|-s|只找来源文件(只找src文件夹)|
|-u|寻找不在以上三个项目中的其他特殊文件|

* PS: 因为 `whereis` 主要是针对 /bin /sbin 底下的执行档，以及 /usr/share/man 底下的 man page 文件，跟几个比较特定的目录来处理。所以速度当然快的多！不过，某些文件是可能找不到

### 通过数据库搜索: locate
用法: `locate [-cilrS] 文件名或目录`

|选项与参数|说明|
|-|-|-|
|-c|不输出文件或目录名，仅计算找到的数量|
|-i|忽略大小写|
|-l|后面接数字，仅输出几行的意思|
|-S|输出 locate 所使用的数据库文件的相关信息，包括该数据库纪录的文件/目录数量等|
|-r|后面可接正规表示法的显示方式|

* PS: `locate` 的搜索需要依靠已建立的数据库 
 /var/lib/mlocate/ , 一般是每天自动更新一次数据库。如果提示数据库没有建立(即"locate: 无法执行 stat () `/var/lib/mlocate/mlocate.db'": 没有那个文件或目录)或没有找到数据，需要执行一下 `updatedb` 建立或更新数据库

* PPS: `updatedb`: 根据 /etc/updatedb.conf 的设定去搜寻系统硬盘内的文件名，并更新 /var/lib/mlocate 内的数据库文件

### 最强大的搜寻命令: find
用法:`find 文件或目录 [option] [action]`

#### 1. 与时间有关的选项, 共有-atime, -ctime 与 -mtime,以 -mtime作为说明
|选项与参数|说明|
|-|-|-|
|-mtime n|n 为数字，意义为在 n 天之前的 24 小时之内被更动过的文件|
|-mtime +n|列出在 n 天之前(不含 n 天本身)被更动过的文件名|
|-mtime -n|列出在 n 天之内(含 n 天本身)被更动过的文件名|
|-mtime file|file 为一个存在的文件，列出比 file 还要新的文件名|
PS: 如果想搜寻24小时内被变更的文件，只需输入 `find / -mtime 0`, 0 代表目前的时间

例子: 
* 想要找出 4 天内被更动过的文件档名？那可以使用`find /var -mtime -4`。那如果是 4 天前的那一天就用` find /var -mtime 4`

* * +4 代表大于等于 5 天前的檔名：ex> find /var -mtime +4
* * -4 代表小于等于 4 天内的文件档名：ex> find /var -mtime -4
* * 4 则是代表 4-5 那一天的文件档名：ex> find /var -mtime 4

#### 2. 使用者或组名有关的参数
|选项与参数|说明|
|-|-|-|
|-uid n|n 为保存在 `/etc/passwd`的用户ID，即UID，搜寻与用户 UID 对应的文件或目录|
|-gid n|n 为保存在 `/etc/group`的组ID，即GID，搜寻与组 GID 对应的文件或目录|
|-user name|name 为保存在 `/etc/passwd`的用户名，搜寻与用户名 name 对应的文件或目录|
|-group name|name 为保存在 `/etc/group`的组名，搜寻与组名 name 对应的文件或目录|
|-nouser|搜寻用户名不存在 `/etc/passwd` 的文件|
|-nogroup|搜寻组不在 `/etc/group` 的文件|
PS: 当你自行安装软件时，很可能该软件的属性当中并没有文件拥有者，这是可能的，在这个时候，就可以使用 -nouser 与 -nogroup 搜寻。

例子：
* 找出 /etc 底下，文件容量大于 50K 且文件所属人不是 root 的档名，且将权限完整的列出 (ls -l)
`find /etc -size +50k -a ! -user root -exec ls -ld {} \;`
PS: ! 代表的是反向选择，亦即不是后面的项目之意

#### 3. 与文件权限及名称有关的参数
|选项与参数|说明|
|-|-|-|
|-name filename|搜寻文件名为 filename 的文件|
|-size [+-]SIZE|搜寻比 SIZE 还要大(+)或小(-)的文件。这个 SIZE 的规格有：c: 代表 byte， k: 代表 1024bytes。所以，要找比 50KB 还要大的文件，就是`find / -size +50k`|
|-type TYPE|搜寻文件的类型为 TYPE 的, 类型主要有：一般正规文件 (f), 装置文件 (b, c), 目录 (d), 连结档 (l), socket (s), 及 FIFO (p) 等属性|
|-perm mode|搜寻文件权限刚好等于 mode 的文件|
|-perm -mode|搜寻文件权限必须要全部囊括 mode 的权限的文件|
|-perm /mode|搜寻文件权限包含任一 mode 的权限的文件|
PS: 利用这个 -name 可以搜寻文件名，默认是完整文件名，如果想要找关键词，可以使用类似 * 的任意字符来处理，即`find / -name '*python*'`

例子:
* 搜寻权限为 -rwsr-xr-x ，亦即 4755 的文件:
`find / -perm 4755`

* 搜寻权限囊括 -rwxr--r-- ，亦即 0744 的文件:
`find / -perm -0744`
PS: 当一个文件的权限为 -rwsr-xr-x ，亦即 4755 时，也会被列出来，因为 -rwsr-xr-x 的属性已经囊括了 -rwxr--r-- 的属性了

* 搜寻文件当中含有 SGID 或 SUID 或 SBIT 的属性:
`find / -perm /7000`

* 搜寻任一权限为 -rwxr-xr-x ，亦即 0755 的文件:
`find / -perm /0755`
PS: 当一个文件属性为 -rw------- 也会被列出来，因为有 -rw.... 的属性存在

#### 4. 额外可进行的动作
|选项与参数|说明|
|-|-|-|
|-exec command|command 为其他指令，-exec 后面可再接额外的指令来处理搜寻到的结果|
|-print|(预设)将结果打印到屏幕上|

例子:
* 找出 /etc 底下，容量大于 1500K 以及容量等于 0 的文件,且将权限完整的列出 (ls -l)
`find /etc -size +50 -o -size 0 -exec ls -l {} \;`
PS: 

关于-exec的说明:
`find 文件名或目录 -exec ls -l {} \;`
* {} 代表的是由 find 找到的内容，结果会被放置到 {} 位置中；
* -exec 一直到 \\; 是关键词，代表 find 额外动作的开始 -exec 到结束 \\; ，在这中间的就是 find 指令内的额外动作。 在本例中就是 `ls -l {}`
* 因为 ; 在 bash 环境下是有特殊意义的，因此利用反斜杠来跳脱。
